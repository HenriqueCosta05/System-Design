# 01. Introdução

Nesse documento, será feita uma introdução sobre os conceitos iniciais de `System Design`, bem como sua importância na Engenharia de Software. Além disso, serão introduzidos `padrões de abordagem em System Design`.

## O que é System Desgin?

`System Design` é definido como o **processo de mapeamento da arquitetura, componentes e interfaces de um sistema**, de forma a corresponder com os requisitos do usuário. Trata-se do processo de arquitetar de soluções de software para lidar com complexidades do mundo real.

## Importância do System Design

Sua importância é notória para qualquer desenvolvedor que queira contruir **aplicações de software robustas, escaláveis e eficientes**. O entendimento dos conceitos é um diferencial para a arquitetura de soluções que lidam com diferentes tipos de complexidades e casos de uso. Alguns dos conceitos que o `System Design` está diretamente envolvido são:

* **Escalabilidade e Confiança**: Os conceitos, se abordados de forma adequada, garantem que os sistemas podem **crescer e lidar com demandas maiores sem falhar**.
* **Gerenciamento eficiente dos recursos**: Ajuda a **otimizar alocação de recursos**, garantindo aplicações rápidas e responsivas.
* **Adaptabilidade**: Permite a criação de sistemas que podem **evoluir com mudanças recorrentes de negócio**, reduzindo custos a longo-prazo.
* **Entendimento arquitetural**: Aprender diferentes arquiteturas de sistemas ajuda na construção de aplicações designadas para diferentes necessidades.

## Como abordar System Design?

O `System Design` pode ser dividido em duas partes complementares:

### 1. High Level Design (HLD)
Essa abordagem define a **arquitetura geral de um sistema e como seus principais componentes se comunicam entre si**. Fornece uma visão geral sobre a arquitetura escolhida para o sistema, serviços e fluxo de dados. É útil para:

* Identificar módulos principais, serviços e suas interações
* Focar na arquitetura do sistema e decisões de alto nível

### 2. Low-Level Design (LLD)
O foco dessa abordagem é nos detalhes da **implementação interna** de cada componente do sistema, descrevendo lógica interna, classes, métodos e estruturas de dados para cada módulo.
